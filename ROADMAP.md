# Roadmap

本文档记录当前产品下一步建设目标，优先服务 Skill 生命周期、权限治理、评审发布与测评集能力完善。

## 1. 完善用户权限机制

目标：建立清晰的用户权限模型，确保发布、评审、编辑等关键操作均受管理权限约束。

重点事项：

- 定义管理权限、编辑权限、评审权限、发布权限之间的关系。
- 对发布、评审、编辑、版本创建、测评集修改等操作增加权限校验。
- 在前端根据用户权限隐藏或禁用不可操作入口。
- 在后端对所有敏感写操作做强制鉴权，避免仅依赖前端控制。
- 补充权限相关测试，覆盖无权限、普通权限、管理权限等场景。

## 2. 支持评审功能

目标：引入 Skill 评审流程，允许用户围绕版本内容进行批注、会签，并在通过后进入发布流程。

重点事项：

- 支持对 SkillVersion 或 bundle 文件发起评审。
- 支持用户在文件、段落或整体版本上添加批注。
- 支持会签记录，包括同意、拒绝、请求修改等状态。
- 定义评审通过条件，例如指定角色或人数完成会签。
- 评审通过后允许发布；未通过或未完成评审时禁止发布。
- 保留评审记录，便于审计与版本追踪。

## 3. 合并历史和概览 Tab

目标：简化 Skill 页面信息结构，将历史记录与概览信息整合，减少页面切换成本。

重点事项：

- 重新设计概览页信息层级，纳入关键历史记录入口。
- 将运行历史、版本活动、审核活动等摘要整合到概览页。
- 保留必要的详情跳转能力，避免概览页过度膨胀。
- 移除或重定向独立历史 Tab，确保导航状态和深链仍可用。

## 4. 完善测评集 Case 能力

目标：增强测评集 Case 的表达能力，支持文件上传等更完整的输入与期望输出场景。

重点事项：

- 支持 Case 上传文件，并将文件作为输入、上下文或期望输出附件。
- 明确文件存储、展示、下载和删除机制。
- 在人工测评和运行结果中展示 Case 附件。
- 支持更丰富的 Case 元数据，例如标签、难度、场景分类。
- 补充后端校验，限制文件大小、类型和数量。
- 补充测评集 Case 的 API、前端表单和持久化测试。
